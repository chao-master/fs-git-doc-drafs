Git Reminders

Git help

Run git help [<command>] for help on a command.

Branch Names

  • Prefix Feature branch names with feature/, this groups them into a feature folder.
  • Do not put /s in branch names, other than the feature/ prefix
  • Only use lower case in branch names, as they are case insensitive
  • Do not use white space in branch names, as this prevents them being easily typed on the command line.
  • Use - instead of spaces in branch names.

Command line

Command line args

  • Single letter options start with one dash (-)
  • Multiple letter (aka word) options start with two dashes (--)
  • The special option -- is used to separate options from file names, all parameters following the -- are treated as file names.
  • Single letter flags (options without and argument), can be immediately followed by another single letter option.
    eg git commit -asv is equivalent to git commit -a -s -v, and git commit -am "Fix capatalization is equivalent to git commit -a -m "Fix capatalization"
  • All single letters options have longer equivalents. I advise writing these in examples as they can make it clearer what is happening.
    eg git commit -a is the short version of git commit --all

Styling

  • Square brackets ([]) indicate something is optional. Pipes (|) inside square brackets indicate you can select zero or one of the options.
    eg -a, --interactive, and --patch are all flags for git commit, but only 0 or 1 of them can be used, so the documentation shows [-a | --interactive | --patch]
  • Regular brackets (()) and pipes are used to indicate that you must use one of the options.
  • Angled brackets (<>) are used to indicate where you place values.
    eg. You would replace all of<msg> the message you want.
  • Ellipses … indicate the previous parameter can be repeated multiple times, separated by spaces.
    eg <file>… allows you to list several files
